Configure the TACTIC Service

For development, test, or evaluation purposes, there is no need to run TACTIC as a service. The startup_dev.py script should suit these purposes adequately. However, for production, it is highly recommended to run TACTIC as a service.

In the <TACTIC_INSTALL_DIR>/src/install directory, there is a directory called service, with two files:

tactic

This file is used for Unix systems and should be copied to the directory where your particular system looks for service executables.

On Fedora Core, for example, this directory is /etc/rc.d/init.d/ (Other flavors of Linux can vary slightly.)

When you copy this file, make sure it is an executable: 

chmod 755 /etc/rc.d/init.d/tactic

And to make it start on reboot, set up chkconfig: 

chkconfig --levels 2345 tactic on

To start it manually, simply type: 

/etc/rc.d/init.d/tactic start
win32_service.py

This script is for running TACTIC as a service on Windows machines.

Simply type: 

python win32service.py install

The TACTIC service should be added to your Windows services repository.

To set up to start automatically on boot, go to:

  • Control Panel > Administrative Tools > Services

  • Click with the right mouse button on "Tactic Application Service" and select Properties.

  • Set the "Startup type" to "Automatic".
